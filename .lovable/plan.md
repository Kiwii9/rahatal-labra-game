## Goal

Let the host choose, per game, whether to use the **built-in question bank** or **author their own questions** (with optional image / video per letter). Custom questions live with the room in the database so guests stay in sync.

---

## User flow

In the Lobby, add a new section: **"مصدر الأسئلة"** (Question source) with two cards:

1. **أسئلة جاهزة** (Built-in) — current behavior, instant start.
2. **أسئلتي الخاصة** (Custom) — opens an authoring panel.

When "Custom" is selected:
- A panel lists all 28 Arabic letters as collapsible rows.
- Each row lets the host add one or more questions: **سؤال** (question text), **الإجابة** (answer), **الفئة** (category, optional), and optional **صورة** (image upload) or **فيديو** (video URL — YouTube/MP4).
- A progress chip shows how many letters have at least one question. Host can start the game once **every letter has ≥1 question** (or skip-letters that won't be used at the chosen grid size).
- Image uploads go to a new `question-media` storage bucket (private-ish, see below). Videos are stored as URLs only.
- The whole question set is saved on the `rooms` row as JSONB.

In the Game:
- `getQuestionForLetter` is replaced with a small helper that picks from `room.custom_questions[letter]` when `question_source = 'custom'`, otherwise falls back to the bundled `QUESTIONS_DB`.
- `QuestionModal` already supports `imageUrl` / `videoUrl`; we just pass them through.

---

## Database changes (one migration)

On `public.rooms`:
- `question_source text not null default 'builtin'` — values `'builtin' | 'custom'`.
- `custom_questions jsonb` — shape: `{ "أ": [{question, answer, category, image_url?, video_url?}], ... }`.

New storage bucket **`question-media`** (public read so guests can render images during a question):
- INSERT/UPDATE/DELETE: only the room host (`auth.uid()` matches the uploader's folder, path layout `<host_uid>/<room_id>/<letter>-<uuid>.<ext>`).
- SELECT: public (same trade-off already accepted for `avatars`).

No changes to RLS on `rooms` — existing "Host can update own room" already covers writing `custom_questions`.

---

## Frontend changes

**New files**
- `src/components/lobby/QuestionSourcePicker.tsx` — the two-card selector.
- `src/components/lobby/CustomQuestionAuthor.tsx` — the per-letter authoring panel (collapsibles + form per question + media upload helper).
- `src/lib/questionResolver.ts` — `resolveQuestion(letter, room)` that returns `{question, answer, category, imageUrl?, videoUrl?}` from either the custom set or `QUESTIONS_DB`.

**Edited files**
- `src/pages/Lobby.tsx` — mount the new components; persist `question_source` + `custom_questions` to the room via `updateRoom`; gate the "إنشاء اللعبة" CTA when source is custom and coverage is incomplete.
- `src/hooks/useRoom.ts` — extend `updateRoom` typing for the two new columns.
- `src/pages/Game.tsx` — replace `getQuestionForLetter(cell.letter)` with `resolveQuestion(cell.letter, roomRow)`; pass `imageUrl`/`videoUrl` into `QuestionModal` (already supported).
- `src/integrations/supabase/types.ts` — auto-regenerated by the migration; do not hand-edit.

**Validation**
- Question text 1–500 chars, answer 1–200 chars, category ≤ 30 chars.
- Image upload: max 3 MB, types `image/png|jpeg|webp|gif`.
- Video URL: must start with `http(s)://`.

**No multi-answer feature, no question editing during a live game** — keeps scope tight.

---

## What is intentionally NOT in this plan

- Reusable / saved question packs across games (could be a follow-up via a `question_packs` table).
- Audio questions (the modal doesn't render audio yet; can be added later).
- Per-question time override from the authoring panel — uses the existing per-question timer set in Lobby.

---

## Acceptance checklist

- Lobby shows the two source cards; "Built-in" preserves today's flow exactly.
- Custom mode lets the host fully author all 28 letters with optional image/video.
- "إنشاء اللعبة" is disabled with a clear message until coverage is complete in custom mode.
- Image uploads succeed and render inside `QuestionModal` for both host and guests.
- Picking a hex during the game pulls the host's custom question for that letter; falls back to built-in when source is `builtin`.
- Guests see images/videos in real time (board sync already routes through `rooms`).
